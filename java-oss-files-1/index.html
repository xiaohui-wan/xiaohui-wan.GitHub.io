阿里OSS文件上传无反应

· 2020-07-03 · # Linux # Java # OSS

问题描述

最近使用阿里OSS文件上传遇到一个问题，采用的是oss的文件上传，走到上传那一步时没有任何反应，本地调试可以，服务器部署后无反应，远程debug服务器也是同样结果，翻阅了oss文档，更换最新的SDK，都是没有任何反应.....

原因

jar包问题，oss的SDK中封装了许多jar包，其中jdom使用的是1.1版本，项目中有部分工具类引入了jdom的2.0版本，导致代码执行到文件上传ossClient.putObject()这一步时没有任何反应(有的人会直接抛出异常提示jar包冲突)。本人在服务器调试时没有任何错误信息......难受~

敲重点

当代码执行到 putObject() 的时候出现任何异常，请记住99%是jar包问题导致的（配置问题除外）.

简述

当然，本人只是说了其中一种问题。如果有异常信息还是要以错误信息去排查问题，不过在解决问题时也学了些新的技能（皮毛）。此次只是记录下这个问题防止以后忘记了

解决问题相关涉及

springboot项目远程debugger调试

  • 远程服务器上启动springboot项目的jar包时设置成可以远程调试的参数命令.

    java -jar -Xdebug -Xrunjdwp:transport=dt_socket,server=y,suspend=n,address=8000 xxx.jar

  • 在IDEA中配置监听

    • 顶部点击 Run 选择 Edit Configurations...
    • 点左上角 + 号，选择 Remote
    • 输入 Host（服务器地址） 、Port （命令中的端口8000），点击 OK 进行保存设置
      注意:个人在实际配置中 Command line arguments for remote JVM:需要配置成 JDK 1.4x 才能够进行远程debug调试。如使用默认JVM配置无法进行操作时，请进行更改配置。
    • 启动方式为 Debug
    • 监听成功

Wan（小提示）

远程debugger调试启动参数命令的说明

  • -Xdebug 是设置远程debugger
  • -Xrunjdwp:transport=dt_socket 是通过socket监听
  • address=8000 是监听的端口号为8000
  • 监听端口与项目的启动端口号没有任何关系，这个不要搞混了
  • 远程服务器代码与本地不相同时优先远程服务器代码，代码不一致时可能会有项目挂掉及其他不可知问题，所以建议调试保持远程服务器代码与本地代码一致
